# StitchWise
**Metric-accurate orthomosaic stitching + aerial disaster damage detection for UAV imagery**

StitchWise is a two-component pipeline for post-disaster aerial analysis:

1. **Stitching** — reconstructs georeferenced orthomosaics from overlapping UAV frames (teammates' component)
2. **Detection** — fine-tunes YOLOv8 on RescueNet to locate water, structural damage, blocked roads, and vehicles in tiled aerial imagery

This README covers the detection component end-to-end.

## Detection pipeline

### Dataset — RescueNet

[RescueNet](https://www.kaggle.com/datasets/yaroslavchyrko/rescuenet) is a post-Hurricane-Michael UAV dataset:
- **4,494 images** at 3000 × 4000 px with pixel-level semantic segmentation masks
- **11 classes** total; we train on 5 disaster-relevant classes (see below)
- Requires a free Kaggle account to download

**Active classes (YOLO indices 0–4):**

| Index | Class |
|-------|-------|
| 0 | water |
| 1 | building-major-damage |
| 2 | building-total-destruction |
| 3 | road-blocked |
| 4 | vehicle |

Excluded: `background`, `building-no-damage`, `building-minor-damage`, `road-clear`, `tree`, `pool` — either uninformative for triage or too visually ambiguous in aerial tiles.

## Step-by-step usage

All scripts resolve paths relative to their own location, so they work regardless of your working directory.

### Step 1 — Prepare the dataset

Converts RescueNet segmentation masks to YOLO bounding-box labels and tiles the 3000 × 4000 px source images into 640 × 640 px patches.

**Quick smoke test (20 source images, ~400 tiles):**

```bash
python detection/prepare_rescuenet.py --img-num 20
```

**Full dataset (4,494 source images, ~90k tiles — takes ~30 min):**

```bash
python detection/prepare_rescuenet.py --img-num 0
```

**If you already have RescueNet downloaded:**

```bash
python detection/prepare_rescuenet.py --dataset-path /path/to/rescuenet --img-num 0
```

**Inspect dataset structure without converting:**

```bash
python detection/prepare_rescuenet.py --inspect-only
```

Output is written to `data/rescuenet_yolo/` with `train/`, `val/`, `test/` splits and a `data.yaml`.

Key flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--dataset-path` | *(auto-download)* | Skip download if already on disk |
| `--mode` | `detect` | `detect` = bounding boxes, `segment` = polygons |
| `--tile` / `--no-tile` | enabled | Tile large images (recommended) |
| `--tile-size` | `640` | Patch size in pixels |
| `--img-num` | `20` | Source images to sample; `0` = full dataset |

---

### Step 2 — Train

Fine-tunes `yolov8m.pt` on the converted dataset using a two-phase strategy: backbone frozen for the first N epochs, then full model fine-tuned end-to-end.

**Quick test run (5 epochs, verifies the pipeline end-to-end):**

```bash
python detection/train.py --epochs 5 --batch 8 --freeze-epochs 0 --name rescuenet_smoke
```

**Standard training run:**

```bash
python detection/train.py --epochs 50 --batch 32 --freeze-epochs 10
```

**Validate setup without starting training:**

```bash
python detection/train.py --dry-run
```

**Resume an interrupted run:**

```bash
python detection/train.py --resume --name rescuenet_detect
```

Best weights are saved to `outputs/runs/rescuenet_detect/weights/best.pt`.
Phase-1 weights (frozen backbone) are saved to `outputs/runs/rescuenet_detect_phase1/`.

Key flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--epochs` | `50` | Total training epochs |
| `--freeze-epochs` | `10` | Epochs with frozen backbone; `0` skips phase 1 |
| `--batch` | `32` | Batch size (`-1` for Ultralytics auto-batch) |
| `--model` | `yolov8m.pt` | Pretrained checkpoint to start from |
| `--name` | `rescuenet_detect` | Run name, controls output subdirectory |
| `--device` | `0` | GPU index or `cpu` |

---

### Step 3 — Evaluate

Runs `model.val()` on the test split and reports mAP50, mAP50-95, per-class Precision / Recall / mAP50, and a confusion summary.

**Standard evaluation on the test split:**

```bash
python detection/evaluate.py
```

**Evaluate on the validation split with a different confidence threshold:**

```bash
python detection/evaluate.py --split val --conf 0.3 --name eval_val
```

**Save sample annotated prediction images for visual inspection:**

```bash
python detection/evaluate.py --visualize --name eval_vis
```

Results are saved to `outputs/runs/eval/`:
- `eval_summary.txt` — plain-text metrics report
- `confusion_matrix.png`, `PR_curve.png` — auto-generated by Ultralytics
- `visualizations/` — annotated images (only with `--visualize`)

Key flags:

| Flag | Default | Description |
|------|---------|-------------|
| `--weights` | `outputs/runs/rescuenet_detect/weights/best.pt` | Checkpoint to evaluate |
| `--split` | `test` | `train`, `val`, or `test` |
| `--conf` | `0.25` | Confidence threshold |
| `--iou` | `0.6` | IoU threshold for NMS |
| `--name` | `eval` | Output folder name under `outputs/runs/` |
| `--visualize` | off | Save up to 16 annotated prediction images |

## Design decisions

### Tiling strategy
Source images are 3000 × 4000 px — far too large for standard YOLO input. Tiling into 640 × 640 patches with 64 px overlap serves two purposes: it brings image resolution into YOLO's native operating range and multiplies effective dataset size (~20× per source image). Only tiles that contain at least one labeled object are saved, which avoids flooding training with background-only patches.

### Aerial augmentation
Standard YOLO augmentations are tuned for street-level imagery. For nadir (straight-down) drone footage:
- **Rotation disabled** — nadir images have no canonical orientation; random rotation produces unrealistic diagonal structures
- **Perspective/shear disabled** — minimal projective distortion in nadir imagery
- **Vertical flip enabled** — there is no "up" when looking straight down, so vertical flip is a valid augmentation
- **Strong mosaic (1.0)** — synthesises varied scenes, especially valuable on a small dataset

### Two-phase training
Training with a frozen backbone for the first 10 epochs lets the randomly-initialised detection head stabilise before gradients flow through the pretrained backbone. Without this, early high-loss gradients can corrupt COCO-pretrained feature representations that would otherwise transfer well to aerial imagery.

### Active class selection
Of the 11 RescueNet classes, 5 were selected based on operational relevance for disaster triage:
- **Retained:** water (flooding extent), building-major-damage, building-total-destruction, road-blocked (route access), vehicle (survivor/responder presence)
- **Dropped:** background, building-no-damage, building-minor-damage, road-clear (these represent the *absence* of a hazard and add noise without operational value), tree and pool (low triage priority)

## Model weights

Trained weights are **gitignored** and must be shared out-of-band (e.g. cloud storage link, shared drive). Place downloaded weights at:

```
detection/model/best.pt
```

or pass the path explicitly:

```bash
python detection/evaluate.py --weights detection/model/best.pt
```
